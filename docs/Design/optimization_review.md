# 策略优化评审报告 (Strategy Optimization Review)

针对 ChatGPT 提出的 12 项建议，我进行了深入的代码核对与技术评估。这是一个非常高水平的 review，指出的问题直击当前模型的痛点。

## 详细分析与评审意见

### 1. 核心改进建议：数据划分策略 (强烈推荐 ⭐⭐⭐⭐⭐)
- **ChatGPT 意见**：目前的随机划分会导致同一盘棋（Session）的照片同时出现在训练集和测试集，造成“数据泄露”，导致测试准确率虚高（95% 可能有水分）。
- **我的意见**：**完全同意**。目前的测试集是在“复习”过往棋谱的噪声，而不是在“预习”全新的环境。应该改为 **按文件夹（Session）划分** 训练/验证/测试集。这能真正测试模型在陌生棋盘、陌生光照下的表现。

### 2. 训练稳定性：双重权重问题 (强烈推荐 ⭐⭐⭐⭐)
- **ChatGPT 意见**：同时使用 `WeightedRandomSampler` 和 `CrossEntropyLoss(weight=...)` 会导致采样和损失计算双重倾斜，引起训练震荡。
- **我的意见**：**非常有道理**。在分类任务中，通常二选一即可。建议保留 **Sampler**（因为它从源头上平衡了 Batch 里的数据分布），删掉 Loss 中的权重。

### 3. Heuristic 逻辑漏洞：反归一化 (关键修复 ⭐⭐⭐⭐)
- **ChatGPT 意见**：启发式过滤在标准化（Normalize）后的 Tensor 上做 `> 1.0` 的判断是不稳定的，容易受不同图片动态范围影响。
- **我的意见**：**确认是 Bug**。我之前的实现确实直接用了标准化后的 Tensor。应该在判断面积前进行 `unnormalize`，或者直接在原始像素空间（0-255）判断。

### 4. 学习率调度：Resume 逻辑 (关键修复 ⭐⭐⭐)
- **ChatGPT 意见**：`CosineAnnealingLR` 在断点续训时如果不重算 `T_max`，LR 曲线会断层。
- **我的意见**：**同意**。我们目前的逻辑是硬编码了总 Epochs。如果我们要更专业的续训，应该动态计算剩余轮数或者使用更鲁棒的 Scheduler。

### 5. 性能上限：Patch Size 与 数据增强 (建议尝试 ⭐⭐⭐)
- **ChatGPT 意见**：48px 略小，建议 64px；增加 `RandomErasing` 对抗高光。
- **我的意见**：**值得一试**。GTX 1080 显存足够撑起平行增加到 64px。高光（Glarel）是围棋识别最大的敌人，`RandomErasing` 是工业界处理此类遮挡/干扰的标准做法。

## 落地改进计划 (Implementation Plan)

我建议分两步走，不要一次性全部改动，以便观察究竟是哪个改动提升了效果。

### 第一阶段：逻辑修复 (立即执行)
1.  **修复数据划分**：修改 `PatchDataset`，改为按 Session 文件夹划分。
2.  **修复双重权重**：移除 `CrossEntropyLoss` 的 `weight` 参数。
3.  **修复启发式逻辑**：在 `get_white_area_ratio` 中加入反归一化步骤。
4.  **修复 LR 调度**：根据 `start_epoch` 动态计算 `T_max`。

### 第二阶段：模型与性能增强 (下一轮实验)
1.  **升级 Patch Size**：将提取尺寸和训练尺寸同步提升至 64px。
2.  **增加增强算子**：引入 `RandomErasing`。

## 第三阶段 (Phase 3): 数据清洗与质量提升 (NEW 🚀)

用户指出数据集中存在大量“脏数据”（Mislabeling/Noise），这是目前制约模型性能进一步提升的根源。

### 1. 为什么会出现脏数据？
*   **定位偏差**：`extract_patches_win.py` 如果找四角定位偏差 1-2 像素，切出来的图块就会位移，边缘可能带上邻近棋子的颜色，导致模型困惑。
*   **传统算法误标**：Gomrade 原本是通过传统 CV 算法标注的，在强反光或极端阴影下，传统算法极易将“光斑”标为白子，将“深影”标为黑子。

### 2. 建议的“数据洗白”方案

#### A. 自动化排查：高损失样本捕捉 (High-Loss Mining)
*   **原理**：用训练好的模型对全量数据跑一遍推理。那些“模型预测结果与标签严重冲突（Loss 极大）”且 **Confidence > 0.9** 的样本，90% 都是标签打错了。
*   **操作**：编写 `identify_dirty_data.py`，将这些“头号嫌疑样本”的路径导出到 CSV，并生成对比大图。

#### B. 文件夹级剔除 (Session Cleaning)
*   **策略**：如果某个棋局文件夹（Session）由于摄像头晃动或遮挡导致定位全错，应直接从 `patches` 中物理删除该 Session 的所有图块，而非试图修复。

#### C. 模型鲁棒性训练
*   **Mixup/Cutmix**：在第二阶段引入这些增强手段，可以强迫模型学习更本质的棋子特征，从而对个别误标样本产生“免疫力”。
