# 围棋棋子分类器 — 训练实验记录 (Progress Log)

本文件记录了在 Windows 环境下进行的模型调参过程、实验改动及结果变化。

---

## 实验记录

### 实验 1: 初始 Baseline (StoneCNN + Sqrt)
- **日期**: 2026-03-03
- **模型架构**: `StoneCNN` (3层卷积)
- **采样策略**: `sqrt` (平方根加权)
- **数据集规模**: 10% 子集
- **结果**:
    - **Accuracy**: 76.9%
    - **表现**: 召回率 (Recall) 尚可，但精确度 (Precision) 较低。模型将大量空位误判为棋子。
    - **结论**: StoneCNN 学习能力有限，且受类别不平衡（空格太多）影响。

### 实验 2: 采样压力测试 (StoneCNN + Equal)
- **日期**: 2026-03-04
- **模型架构**: `StoneCNN`
- **采样策略**: `equal` (1:1:1 强制平衡)
- **结果**:
    - **Accuracy**: 58.0% (退步)
    - **表现**: 召回率提高到 99%（几乎不漏子），但误报率爆炸（一半空位被看成棋子）。
    - **结论**: StoneCNN 的“脑容量”不足以通过学习纹理区分交叉点，只能靠猜概率。

### 实验 3: 架构升级 (MobileNetV3 + Sqrt)
- **日期**: 2026-03-04
- **模型架构**: `MobileNetV3` (工业级轻量化网络)
- **采样策略**: `sqrt` (回归更平衡的策略)
- **结果**:
    - **Accuracy**: 90.0% (重大突破)
    - **表现**: 召回率 (Recall) 保持在 91% 的高位，同时精确度 (Precision) 从 30% 翻倍提升到了 **60% 左右**。误报率大幅下降。
    - **结论**: 验证了核心瓶颈在于模型表达能力。MobileNetV3 能够有效学习棋子与棋盘特征，即使在只有 10% 数据的情况下也能达到商用初步门槛。

---

### 实验 4: 磁盘寻道优化测试 (MobileNetV3 + 30% Data)
- **日期**: 2026-03-04
- **模型架构**: `MobileNetV3`
- **数据集规模**: 30% 数据 (约 12 万张 Patch)
- **改动说明**: 
    1. 发现 100% 数据 (40万) 时机械硬盘出现卡死，怀疑是磁头寻道压力过大。
    2. 将 `num_workers` 从 4 降为 2，减少磁盘 I/O 冲突。
    3. `subset_ratio` 调整为 `0.3`。
- **结果**:
    - **Accuracy**: 90.0% (总体准确率持平)
    - **表现**: 
        - **黑棋 (B)**: Precision 提到到了 78% (对比实验3翻倍)，Recall 94%。
        - **白棋 (W)**: Precision 依然只有 46%，Recall 97%。
        - **空位 (E)**: Recall 89%，约 11% 的空位被误判（绝大部分被误判成了白棋）。
    - **结论**: 数据量增加到 30% 显著提升了黑棋的辨识度。白棋的低精确度说明模型依然分不清“亮色棋盘/交叉点”和“白棋”。HDD 在 30% 数据下配合 2 个 Worker 跑得比较稳。

### 实验 5: 断点续训 + 尺寸启发式修正 (Resume + Heuristic)
- **日期**: 2026-03-04
- **架构**: `MobileNetV3` (继续使用)
- **数据量**: 60% (续训，增加 30% 新数据)
- **改动说明**: 
    1. 使用 `--resume` 从实验 4 的断点继续。
    2. 引入“白子面积过滤”小手段：在模型预测为 W 时，检查图片中高亮区域占比。若占比过小（<10%），则修正为 E（空位）。
    3. 目的：通过物理先验（棋子肯定比光点大）强行压低白子的误报。
- **结果**:
    - **Accuracy**: **95.0%** (重大突破)
    - **表现**: 
        - **白棋 (W)**: Precision 从 46% 飙升至 **90%**！Recall 为 72%。
        - **黑棋 (B)**: Precision 77%，Recall 94%。
        - **空位 (E)**: Precision 97%，Recall 97%。
    - **结论**: “白子面积过滤”小手段效果极其显著，准确杀掉了绝大部分光点误报。虽然白棋的 Recall (召回率) 略有下降（说明有些白棋被误杀成了空位），但整体 95% 的准确率已经达到了非常高的可用水平。

### 实验 7: 清洗数据 + Session 物理隔离 + RAM 优化 (Current Run)
- **日期**: 2026-03-05
- **架构**: `MobileNetV3`
- **数据量**: 100% 清洗后数据 (已移除 4,679 个脏数据)
- **改动说明**: 
    1. **物理隔离**: 引入了真正的 Session 分隔（按文件夹划分训练/验证集），彻底杜绝了数据泄露。
    2. **RAM 预加载**: 所有图片进内存，解决 HDD 瓶颈。
    3. **数据清洗**: 移除了高置信度误标样本。
- **当前状态 (10/100 Epochs)**:
    - **Train Acc**: 88.13% (稳步上升)
    - **Val Acc**: 68.54% (相对平稳)
- **分析**:
    - **20% 的巨大差额**: 这是由于引入了 Session 隔离带来的“真实准确率”。以前 95% 的准度是因为验证集里有和训练集极其相似的同一盘棋的画面（数据泄露）。现在的 68% 反映了模型在面对“全新灯光、全新棋盘”时的真实泛化能力。
    - **性能稳定性**: Epoch 8 出现了 5.66 batch/s 的掉速，可能是系统后台活动或显卡降频导致。

## 指标解读小课堂
- **Precision (精确率)**: 判为黑棋的里面，有多少是真的黑棋？(高 Precision = 误报少)
- **Recall (召回率)**: 全部的黑棋里面，你找回了多少个？(高 Recall = 漏判少)
- **class E (空位)**: 高 Precision 意味着你报它是棋子时很准；高 Recall 意味着你没有漏看任何空位。
