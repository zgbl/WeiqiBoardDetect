# Checkpoint v7: Stability & RAM Baseline (Interrupted)
================================================================

### 🏁 训练重点 (Focus)
*   **工程稳定性**：解决 Windows 下训练频繁挂起或等待过久的问题。
*   **内存优化**：首次尝试大规模 RAM 预加载（RAW 模式），跳过 HDD IO 瓶颈。
*   **混合训练**：将 Dataset1 和 Dataset2 混合进行大规模 Epoch 训练（原计划 100 轮）。

### 🛠 主要修改 (Key Changes)
*   **Windows 多进程修复**：强制在 RAM 模式下设置 `num_workers=0`，彻底解决了子进程导致的内存溢出和卡顿。
*   **RAM Preloading**：代码重构，支持启动时一次性读入所有图块到内存。
*   **初步存档**：建立了 `train_win_ram.py` 的基础架构。

### 📊 训练结果评价 (Results)
*   **状态**：中断 (Interrupted)。
*   **表现**：Train Acc 冲到 ~88%，Val Acc 停留在 ~68%。
*   **观察**：明显的“天花板”效应。训练集和验证集的巨大鸿沟暴露了数据分布不一致的问题。

### ⚠️ 存在问题与反思 (Issues)
*   **数据污染预兆**：数据集混合后，脏数据开始稀释优质数据。
